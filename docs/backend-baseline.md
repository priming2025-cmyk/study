## 백엔드 베이스라인: Supabase (MVP 최소 서버)

### 선택 이유(요약)
- **Auth + Postgres + RLS + Realtime**를 즉시 사용 가능해 초기 서버 개발/운영비가 낮음
- 학생/부모/교사(기관) 권한 분리를 **RLS 정책**으로 강제할 수 있어 민감도 높은 서비스에 유리
- 스터디방(WebRTC)에서 필요한 **시그널링/상태 동기화**를 Realtime 채널로 구현 가능

### 서버가 맡는 일(MVP)
- 인증/역할: 학생/부모(향후 교사/기관)
- 데이터 저장: 계획표, 학습 세션 요약, 부모-자녀 링크, 열람 로그
- 시그널링: 스터디방 SDP/ICE 교환(미디어는 P2P로 직접)
- 통계: “주간 요약” 정도만 최소 집계(가능하면 클라이언트 렌더 중심)

### 서버가 맡지 않는 일(비용/개인정보 절감)
- 얼굴인식/집중도 추정(카메라 분석): **전부 온디바이스**
- 카메라 프레임/영상 저장/전송: **금지**

### Realtime 채널 범위(MVP)
- `study_room:<room_id>`: 방 참여/퇴장, SDP offer/answer, ICE candidate, 공부중/휴식중 상태 브로드캐스트
  - 원칙: 메시지는 짧게, TTL/로그 최소화(가능하면 DB 영구 저장 X)
